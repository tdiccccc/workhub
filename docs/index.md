# Docs

## 概要

本ディレクトリは **WorkHub** の設計・仕様・学習内容を管理するドキュメントです。

WorkHubは単なる個人開発ではなく、**Java + Spring Boot + Nuxt4 を体系的に学習し、実務レベルの開発プロセスを経験すること**を目的としています。

そのため、本プロジェクトでは要件定義から設計、実装、レビュー、学習記録までをドキュメントとして管理します。

---

# ディレクトリ構成

```text
docs/
├── api/              # API仕様書
├── architecture/     # システム構成・アーキテクチャ
├── database/         # ER図・テーブル定義
├── learning/         # 学習ログ
├── milestones/       # 開発計画・リリース計画
├── requirements/     # 要件定義
├── route/            # フロントエンド画面ルート定義
└── screen/           # 画面設計
```

---

# ドキュメント一覧

| ディレクトリ          | 内容                                    |
| --------------- | ------------------------------------- |
| `requirements/` | システム要件、機能要件、非機能要件を管理します。              |
| `architecture/` | システム構成、アーキテクチャ、ディレクトリ構成、採用技術などを管理します。 |
| `database/`     | ER図、テーブル定義、データベース設計を管理します。            |
| `api/`          | REST API仕様、認証方式、レスポンス仕様などを管理します。      |
| `route/`        | Nuxt4の画面ルーティング、画面遷移を管理します。            |
| `screen/`       | 画面設計、UI設計、画面レイアウトを管理します。              |
| [`milestones/`](./milestones/index.md) | MVPから正式リリースまでの開発計画を管理します。             |
| `learning/`     | 学習内容、設計上の気付き、実装メモを記録します。              |

---

# ドキュメント管理方針

* ドキュメントを先に作成し、設計内容を明確にしてから実装を開始する。
* 完成形を一度に実装せず、MVPから段階的に機能を追加する。
* API・画面・データベースの設計は常に整合性を保つ。
* 設計変更が発生した場合は、実装より先にドキュメントを更新する。
* 学習で得た知見は `learning/` に記録し、継続的にナレッジを蓄積する。

# 開発方針

WorkHubは「完成したら終わり」のプロジェクトではありません。

MVPを完成させた後も、GitHub Milestoneに沿って継続的に機能追加・リファクタリングを行い、実務で通用するアプリケーションへ育てていきます。

また、本プロジェクトはポートフォリオであると同時に、Java・Spring Boot・Nuxt4・AWSを体系的に学ぶための教材としても位置付けています。
