## フロントエンド実装方針

v0.1 では、画面デザインを作り込みすぎず、Nuxt4 と Spring Boot API の接続、認証状態管理、Project CRUD の操作完了を優先する。

Tailwind CSS は UI/UX 設計が固まってから導入する。v0.1 前半では、素の Vue/Nuxt 実装で画面遷移、フォーム、API 通信、状態管理を理解する。

### 実装順序

1. ログアウト API を実装する
2. ログイン中ユーザー取得 API を実装する
3. Pinia で認証状態を管理する
4. Nuxt middleware で未ログイン時の遷移制御を行う
5. Zod Schema でログインフォームと Project フォームのバリデーションを行う
6. UI/UX 設計を整理する
7. Tailwind CSS を導入する

### 認証 API

```text
POST /api/auth/logout
GET  /api/auth/current-user
```

`POST /api/auth/logout` はサーバー側のセッションを破棄する。

`GET /api/auth/current-user` はログイン中ユーザー情報を返す。未ログインの場合は 403 を返す。

### フロントエンド認証状態管理

```text
stores/auth.ts
  user
  isLoggedIn
  fetchCurrentUser()
  login()
  logout()

middleware/auth.ts
  未ログインの場合は /login へ遷移する
```

### Tailwind CSS 導入方針

Tailwind CSS は、認証状態管理とフォームバリデーションの流れができた後に導入する。

導入時は、まず以下の最小範囲に限定する。

- 余白
- フォームの視認性
- ボタンの見分けやすさ
- 画面全体の最低限のレイアウト
